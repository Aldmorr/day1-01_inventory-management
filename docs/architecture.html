Factory Inventory Management

A full-stack inventory and operations dashboard. Vue 3 + Composition API on the frontend, FastAPI on the backend, with in-memory JSON-file data — no database. Designed as a Claude Code workshop reference app.

1Tech Stack

Modern stack with reactive frontend, type-safe backend, and zero database overhead.

Frontend — client/

Single-page Vue 3 app served by Vite dev server on port 3000.
  • Vue 3.4.21 — Composition API throughout
  • Vue Router 4.3.0 — 6 routes, one per view
  • Vite 5.2.0 — dev server + HMR
  • Axios 1.6.7 — HTTP client in api.js
  • State: refs + computed + a singleton composable (useFilters) — no Pinia/Vuex
  • i18n: hand-rolled useI18n() with en.js / ja.js

Backend — server/

FastAPI service exposing a JSON REST API on port 8001.
  • FastAPI ≥ 0.110 — endpoints in main.py
  • Uvicorn ≥ 0.24 — ASGI server
  • Pydantic ≥ 2.5 — request/response models
  • Tests: pytest + httpx + TestClient
  • Data layer: mock_data.py loads 7 JSON files at startup into module globals
  • CORS: allow_origins=["*"] (dev only)

4Data Flow — One Request

Following a single user action: picking "San Francisco" in the warehouse filter on the Dashboard.

User changes the dropdown in the global filter bar. FilterBar.vue
Singleton ref updates via the useFilters() composable — any view subscribed to it reacts. composables/useFilters.js
Dashboard re-fetches by calling getDashboardSummary(filters). views/Dashboard.vue
Axios issues HTTP GET http://localhost:8001/api/dashboard/summary?warehouse=San%20Francisco&…. api.js
FastAPI route handler runs, calls apply_filters() against the in-memory inventory + orders lists. server/main.py
Aggregates computed: inventory value, low-stock count, pending orders, backlog, revenue.
Pydantic serializes the response dict to JSON.
Dashboard binds the result to summary ref → computed properties recompute → KPI cards re-render.
